About Minecraft Plugin Development

huanmoon007
2024-03-30

(By the way this article has been finished for like 1 month but for some reasons I publicise it today)

So recently, I’m working on some minecraft plugins. I used to develop some minecraft mods, and minecraft pvp clients (which are reeeeeaaaaaaaally stupid).

At first, I was just playing a game called housing on hypixel, and the thing you need to do is coding and building your housing and invite your friend to the stupid housing you made, and get a pointless stuff called cookies

I mean, you can’t even eat the cookies, you can’t even use it to buy stuff, and what the hack can it do???

In order to get cookie in real life (I’m starving bro), I need to make money first and then buy them, I started developing the ULTIMATE MINECRAFT PLUGIN and put it on my server and then

I CAN MAKE SOME STUFF LIKE VIP AND FINALLY MAKE MONEY YEEEEEEEEY! LET’S GO!!!!!!

and that’s the reason why I decided to develop minecraft plugins.

then let’s talk about the differences between plugin development and mod development.

the plugin can only modify things on server side, but not the client side, that means if you want to make a gui, you need to make an inventory which contains some items with specific name as the button of the gui menu

(I don’t think call it gui is a good idea, it should be chest ui actually, but community just called it gui)

compare to plugins, mods are much more convenient, you can modify things on client side, and make a real gui menu, new particles, new textures and etc.

However, I still prefer to plugins because to use plugins, you don’t need to install ANYTHING, you can just join a server and enjoy the plugin

I’m a minimalist, so I love the feeling of playing game without installing a lot of mods which can make your game LLLLLAAAAAGGGG (I love lightweight client-side mods though)

what about edit vanilla content? which one is more convenient and easy, mods or plugins?

well, frankly speaking, mods are convenient, there’s a stuff called ‘mixin’ in mod development, and developer can just use it to inject to the vanilla game

However, for plugins, developers must use things like listeners and nms to modify vanilla contents

sadly, nms are EXTREMELY HARD the methods in it are like ‘NMSPLAYER.f(int a)’ bro how can I know what do f and a stand for??????

but just like I mentioned above, it’s hard but I still prefer to plugins

wait maybe I’m not only a minimalist, but also a
